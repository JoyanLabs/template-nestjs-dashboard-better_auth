## ⭐️ Pasos

1. Usa el tracker de issues para asegurarte de que la solicitud de funcionalidad o el bug no haya sido reportado ya 🔎.
2. Envía un issue describiendo tu cambio propuesto al repositorio 💡.
3. El propietario del repositorio responderá a tu issue tan pronto como podamos 💪.
4. Si tu cambio propuesto es aceptado, haz fork del repositorio, desarrolla y prueba tus cambios de código 🤝.
5. Asegúrate de que tu código se adhiera al estilo existente en el código 💅🏻.
6. Títula tu pull request siguiendo el estilo de [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) 🪄.
